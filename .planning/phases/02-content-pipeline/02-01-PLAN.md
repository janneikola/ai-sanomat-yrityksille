  <action>
    Install dependencies: `npm install rss-parser node-cron && npm install -D @types/node-cron` (run from api/ directory).

    **api/src/integrations/rssCollector.ts:**
    Create RSS feed fetcher using rss-parser library. Export `fetchRssFeed(url: string)` that returns `Array<{ title: string; url: string; summary: string | null; publishedAt: Date | null }>`. Set 15-second timeout per feed. Set User-Agent header to 'AI-Sanomat-Collector/1.0'. Map `item.title`, `item.link`, `item.contentSnippet` (fallback to `item.content`), and `item.pubDate`. Filter out items with empty/missing URLs.

    **api/src/integrations/beehiivClient.ts:**
    Create Beehiiv API v2 client. Export `fetchBeehiivPosts(publicationId: string, apiKey: string, limit?: number)` returning the same array shape as rssCollector. Use `https://api.beehiiv.com/v2/publications/${publicationId}/posts?status=confirmed&limit=${limit}&order_by=publish_date&direction=desc`. CRITICAL: Beehiiv `publish_date` is Unix seconds -- multiply by 1000 for JavaScript Date. Map `post.title`, `post.web_url`, `post.subtitle` (as summary). Add Bearer auth header from apiKey param.

    **api/src/services/newsCollectorService.ts:**
    Create the orchestration service following the existing service pattern (import db from '../db/index.js', schema from '../db/schema.js'). Export `collectAllNews()` that:
    1. Fetches all active sources: `db.select().from(newsSources).where(eq(newsSources.isActive, true))`
    2. Iterates each source in a for loop (NOT Promise.all -- sequential to avoid rate limits)
    3. For RSS sources (type === 'rss' and url exists): calls `fetchRssFeed(source.url)`
    4. For Beehiiv sources (type === 'beehiiv' and config exists): parses config JSON for publicationId, calls `fetchBeehiivPosts(publicationId, process.env.BEEHIIV_API_KEY!)`
    5. For each collected item, inserts into newsItems using `db.insert(newsItems).values({ sourceId: source.id, title, url, summary, publishedAt }).onConflictDoNothing()` -- silent deduplication on URL unique constraint
    6. Each source is wrapped in try/catch -- log error with `console.error(\`Failed to collect from ${source.name}:\`, error)`, increment error counter, continue to next source
    7. Returns `{ collected: number, errors: number, sources: number }`

    **api/src/scheduler.ts:**
    Create scheduler using node-cron. Export `startScheduler()` that registers a cron job at `'0 6 * * *'` (daily at 06:00) with `timezone: 'Europe/Helsinki'`. The job calls `collectAllNews()` and logs results. Also export `triggerCollection()` that directly calls `collectAllNews()` (for API-triggered collection).

    **api/src/app.ts:**
    Add `import newsRoutes from './routes/news.js'` and register: `await app.register(newsRoutes, { prefix: '/api/admin' })`. Also add `import { serveImages } from './plugins/static.js'` -- NO, skip static serving for now (images are Phase 2 Plan 02 concern). Just add the news routes.

    **api/src/index.ts:**
    Add `import { startScheduler } from './scheduler.js'` and call `startScheduler()` after `app.listen()` succeeds. This starts the daily cron job when the server boots.
