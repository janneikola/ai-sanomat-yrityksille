**3. Create xClient.ts (api/src/integrations/xClient.ts):**
Direct HTTP calls to Apify Tweet Scraper V2 sync endpoint. Follow tavilyClient.ts lazy-init pattern.

- Env var: `APIFY_TOKEN` (not APIFY_API_KEY -- match Apify official convention)
- Actor URL: `https://api.apify.com/v2/acts/apify~tweet-scraper-v2/run-sync-get-dataset-items`
- `timeout=120` query param (2 minutes)
- Export `ApifyTweet` interface with fields: id, text, createdAt, likeCount, retweetCount, replyCount, quoteCount, bookmarkCount, isRetweet, isReply, author (userName, name, profileImageUrl), url
- Export `fetchTweetsByHandle(handle: string, maxItems = 20): Promise<ApifyTweet[]>` -- uses `handles: [handle]` input
- Export `searchTweets(searchTerms: string[], maxItems = 50): Promise<ApifyTweet[]>` -- uses `searchTerms` input
- Both include `proxyConfig: { useApifyProxy: true }` in body
- If APIFY_TOKEN not set: log warning and return empty array (graceful fallback, never crash)

**4. Create xCollectorService.ts (api/src/services/xCollectorService.ts):**
Handles influencer timeline collection for x_account sources.

- Export `collectXAccounts(): Promise<{ collected: number; errors: number }>`:
  - Query all active x_account sources from newsSources
  - For each source, parse config JSON as `{ handle: string; description?: string; includeReplies?: boolean; minLikes?: number }`
  - Call `fetchTweetsByHandle(config.handle, 20)`
  - Filter tweets: skip if `tweet.isRetweet === true`, skip if `tweet.isReply === true` and `!config.includeReplies`, skip if `config.minLikes` and `tweet.likeCount < config.minLikes`
  - Normalize tweet URL: `https://x.com/${tweet.author?.userName}/status/${tweet.id}` (strip query params, always x.com format)
  - Map to news_items shape: title = `tweet.text.slice(0, 100) + (text.length > 100 ? '...' : '')`, url = normalized URL, summary = full tweet text, publishedAt = `new Date(tweet.createdAt)`
  - Insert with `onConflictDoNothing` (URL unique constraint handles dedup)
  - Track health via existing `logFetchAttempt` and `updateSourceHealth` for each source
  - On error per source: log, increment errors counter, call `checkAutoDisable` and `checkHealthTransitionNotification`
  - After each successful fetch: call `recordBudgetUsage('influencer', source.id, tweets.length)`
  - Check budget before each fetch: call `checkBudget()`, if remaining <= 0, log warning but continue (soft cap per user decision)

**5. Create xSearchService.ts (api/src/services/xSearchService.ts):**
On-demand per-client keyword search, modeled after webSearchService.ts.

- Export `searchXForClient(clientId: number): Promise<{ collected: number; queries: number }>`:
  - Query active x_search sources where config contains the client's ID
  - For each source, parse config as `{ query: string; language?: string; clientId?: number }`
  - Call `searchTweets([config.query], 50)`
  - Apply engagement threshold filter: `tweet.likeCount >= 5 || tweet.retweetCount >= 2` (Claude's discretion -- low threshold to avoid losing breaking news)
  - Skip retweets and replies (same filter as influencer)
  - Normalize URLs and map to news_items (same as xCollectorService)
  - Insert with onConflictDoNothing
  - Track health per source via sourceHealthService
  - Track cost via `recordBudgetUsage('search', source.id, tweets.length)`

**6. Create xBudgetService.ts (api/src/services/xBudgetService.ts):**
Budget tracking and cost estimation.

- Cost formula: `(tweetCount / 1000) * 0.40` ($0.40 per 1,000 tweets)
- Budget limit from env: `Number(process.env.X_MONTHLY_BUDGET ?? '50')` (default $50/month)
- Budget warning threshold: 80% (Claude's discretion)
- Export `recordBudgetUsage(runType: 'influencer' | 'search', sourceId: number, tweetsCollected: number): Promise<void>`:
  - Insert row into xBudgetUsage with current month ('YYYY-MM'), estimated cost, tweet count
- Export `checkBudget(): Promise<{ spent: number; limit: number; remaining: number; warningLevel: 'none' | 'warning' | 'exceeded' }>`:
  - Sum estimatedCost from xBudgetUsage for current month
  - warningLevel: 'warning' at >= 80%, 'exceeded' at >= 100%
  - Log warnings at 80% and 100% thresholds
- Export `getBudgetSummary(): Promise<{ currentMonth: { spent, limit, remaining, warningLevel, tweetsCollected }; history: Array<{ month, spent, tweetsCollected }> }>`:
  - Returns current month summary plus last 6 months history for dashboard display
