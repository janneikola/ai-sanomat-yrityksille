**4. Create `api/src/services/emailService.ts`:**
Two exported functions:

`renderDigestEmail(issue, client)` -- renders the React Email template to HTML + plain text:
- Parse issue.generatedContent as DigestContent (with imageUrls on stories)
- Construct absolute image URLs: use `process.env.PUBLIC_URL || 'http://localhost:3000'` as base, prepend to heroImageUrl and story imageUrls
- Construct unsubscribeUrl placeholder (will be per-member when sending)
- Use `render()` from `@react-email/render` to produce HTML
- Use `render({ plainText: true })` to produce plain text
- Return `{ html, text }`

`sendDigestToClient(issueId)` -- orchestrates sending to all members:
- Fetch issue by ID (must be status 'approved')
- Fetch client by issue.clientId
- Fetch all active, non-bounced members for the client: `where(eq(members.clientId, issue.clientId))` + `where(eq(members.isActive, true))` + `where(eq(members.isBounced, false))`
- Render email HTML+text using renderDigestEmail
- Build email payloads per member:
  - `from`: `'AI-Sanomat <noreply@mail.aisanomat.fi>'`
  - `to`: member.email
  - `subject`: `AI-Sanomat viikkokatsaus: ${client.name}`
  - `headers`: both List-Unsubscribe and List-Unsubscribe-Post per RFC 8058
    - `'List-Unsubscribe': '<https://app.aisanomat.fi/api/unsubscribe?member=${member.id}>'`
    - `'List-Unsubscribe-Post': 'List-Unsubscribe=One-Click'`
- Call sendBatchEmails
- Create deliveryStats records per member with the returned resendMessageId, sentAt = new Date(), status = 'sent'
- Update issue status to 'sent'
- Return `{ sent: members.length, issueId }`

**1. Create `api/src/routes/webhooks.ts`:**
Public webhook route (NO authenticate middleware) for Resend events:

- Register `fastify-raw-body` plugin with `{ global: false }` at the route plugin level
- `POST /webhooks/resend` with `config: { rawBody: true }`:
  - Verify signature using `svix` Webhook class: `new Webhook(process.env.RESEND_WEBHOOK_SECRET).verify(rawBody, headers)`
  - Extract svix-id, svix-timestamp, svix-signature from request headers
  - Type the event payload as `ResendWebhookEvent` (define interface: type, created_at, data with email_id, from, to[], subject, optional bounce with message/type/subType)
  - Process by event type:
    - `email.delivered`: update deliveryStats.status = 'delivered' where resendMessageId matches email_id
    - `email.opened`: update deliveryStats status = 'opened' and openedAt = new Date() where resendMessageId matches
    - `email.bounced`: update deliveryStats status = 'bounced' and bouncedAt = new Date(). If bounce.type === 'Permanent', also set members.isBounced = true (look up memberId from the deliveryStats record)
  - Return 200 `{ received: true }`
  - If no matching deliveryStats record found for email_id, silently return 200 (don't fail)
  - If signature verification fails, return 400

**2. Extend `api/src/routes/digests.ts`:**
Add these new endpoints to the existing digest routes file (keep existing routes intact):

- `GET /digests/:id/preview` (authenticated):
  - Fetch issue, verify it exists and has generatedContent
  - Render HTML using emailService.renderDigestEmail
  - Return `reply.type('text/html').send(html)` -- raw HTML for iframe preview

- `POST /digests/:id/approve` (authenticated):
  - Fetch issue, verify status is 'ready'
  - Update status to 'approved'
  - Return 200 with updated issue

- `POST /digests/:id/send` (authenticated):
  - Fetch issue, verify status is 'approved'
  - Call emailService.sendDigestToClient(issueId)
  - Return 200 with send result `{ sent: number, issueId: number }`

- `POST /digests/:id/regenerate` (authenticated):
  - Fetch issue, verify status is 'ready' or 'approved' or 'failed'
  - Call newsletterService.generateClientDigest(issue.clientId) -- reuses existing pipeline
  - Return 201 with new issue result
