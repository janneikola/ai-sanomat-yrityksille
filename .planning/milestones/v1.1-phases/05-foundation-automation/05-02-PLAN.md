- Query `sourceHealthLogs` table where `sourceId` matches, order by `fetchedAt` DESC, limit to `limit` entries
- Return array of `{ id, success, itemCount, errorMessage, fetchedAt }`

**2. Source Routes Extension (api/src/routes/sources.ts):**

Modify `GET /sources` to:
- Accept optional query parameter `healthStatus` (enum: 'green' | 'yellow' | 'red')
- If `healthStatus` param is provided, filter the list to only sources matching that computed status
- Response schema already includes health fields from Plan 01's shared schema update

Add `GET /sources/:id/health-logs` route:
- Schema: params `{ id: z.coerce.number() }`, query `{ limit: z.coerce.number().default(20) }`
- Response: array of `{ id: number, success: boolean, itemCount: number, errorMessage: string | null, fetchedAt: string }`
- Handler calls `getSourceHealthLogs(id, limit)`
- Auth: `fastify.authenticate`

**3. Dashboard Routes Extension (api/src/routes/dashboard.ts):**

Add `GET /dashboard/scheduler-runs` route:
- Query `schedulerRuns` table, order by `startedAt` DESC, limit 30
- Response schema: array of `{ id, startedAt, completedAt, clientsProcessed, successes, failures, skips, notes }`
- Auth: `fastify.authenticate`

Modify `GET /dashboard/stats` to include `nextScheduledDate` per client:
- After building each client's stats, compute `nextScheduledDate` using `getNextScheduledDate()` from scheduleService
- Only compute if `schedulePaused === false`; otherwise return `null`
- Add to response: `nextScheduledDate: z.string().nullable()`, `scheduleFrequency: z.string()`, `schedulePaused: z.boolean()`
- Update the response Zod schema to include these new fields
