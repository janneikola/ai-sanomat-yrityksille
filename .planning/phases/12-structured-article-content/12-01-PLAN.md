**IMPORTANT rendering decisions:**
- Use `story.lead` as the condition for structured vs fallback rendering. If `lead` exists, render structured content. If not, fall back to `businessImpact`.

- Do NOT render `businessImpact` when structured content is present -- the prompt instructs Claude to make `businessImpact` a brief fallback summary, so showing both would be redundant.
- Use `<Heading as="h3">` instead of `<Text>` for story titles to give proper semantic hierarchy (CONTENT-03).
- Use inline `style={}` objects (not className) for the structured content elements. This ensures Outlook Word engine compatibility. The fallback `businessImpact` can keep existing Tailwind classes since that path is only for old digests.
