    6. Returns the public URL path: `/images/{uuid}.png`
    7. Returns null if no image data in response

    Use `IMAGE_STORAGE_PATH` from env (default: `./uploads`). Use `node:crypto` for randomUUID, `node:fs` and `node:path` for file operations.

    **api/src/services/imageService.ts:**
    Export `generateDigestImages(imagePrompts: ImagePrompts): Promise<{ heroUrl: string; sectionUrls: string[] }>` that:
    1. Generate hero image: `generateImage(imagePrompts.heroPrompt, 1200, 630)`. On failure or null result, use `PLACEHOLDER_IMAGE_URL`.
    2. Generate section images SEQUENTIALLY (not Promise.all -- avoid Gemini rate limits per research pitfall #4): for each sectionPrompt, call `generateImage(prompt, 800, 450)`. On failure, use `PLACEHOLDER_IMAGE_URL`.
    3. Each generateImage call is wrapped in its own try/catch -- one failure does not prevent others.
    4. Returns `{ heroUrl, sectionUrls }` where each URL is either the generated image path or the placeholder.

    **api/src/services/newsletterService.ts:**
    Export a template helper: `fillTemplate(template: string, variables: Record<string, string>): string` that replaces `{{variableName}}` patterns with values.

    Export `generateClientDigest(clientId: number): Promise<{ issueId: number; status: string }>` that orchestrates the full pipeline:

    1. **Get client**: `db.select().from(clients).where(eq(clients.id, clientId))`. Throw if not found.
    2. **Get recent news items**: `db.select().from(newsItems).orderBy(desc(newsItems.collectedAt)).limit(30)` -- limit to 30 most recent per research pitfall #7. Format as text: title + summary for each.
    3. **Get prompt templates**: Fetch all three by name ('viikkokatsaus_generointi', 'faktojen_validointi', 'kuvapromptit') using `db.select().from(promptTemplates).where(eq(promptTemplates.name, name))`.
    4. **Create issue record**: Insert into issues with `{ clientId, weekNumber: getWeekNumber(), year: new Date().getFullYear(), status: 'generating' }`. Helper `getWeekNumber()` computes ISO week number.
    5. **Generate digest**: Fill the generation template with `{ industry: client.industry, company_name: client.name, news_items: formattedNews, previous_issues: '' }`. Call `generateDigest(filledTemplate, 'Generate the digest')`. Use the filled template as the system prompt and a short user prompt like 'Generoi viikkokatsaus' (Generate the weekly digest).
    6. **Update issue**: Set `generatedContent: JSON.stringify(digest)`, `status: 'validating'`.
    7. **Validate digest**: Fill the validation template with `{ generated_digest: JSON.stringify(digest), source_articles: formattedNews }`. Include the humanizer skill's 26 AI-pattern rules in the system prompt by appending a section that lists the pattern categories (passive overuse, missing particles, translation structures, genitive chains, significance inflation, flattering tone, etc.) and instructs the validator to check for them. Call `validateDigest(filledSystemPrompt, 'Validate the digest')`.
    8. **Update issue**: Set `validationReport: JSON.stringify(validationReport)`.
    9. **Generate image prompts**: Fill the image prompts template with `{ digest_sections: JSON.stringify(digest.stories), industry: client.industry }`. Call `generateImagePrompts(filledTemplate, 'Generate image prompts')`.
    10. **Generate images**: Call `generateDigestImages(imagePrompts)`.
    11. **Update issue**: Set `heroImageUrl: images.heroUrl`, store section image URLs in the digest content by updating each story with an `imageUrl` field (modify the generatedContent JSON), set `status: 'ready'`.
    12. On ANY error during steps 5-11: update issue status to 'failed', log the error, re-throw.

    Wrap the entire pipeline in a try/catch that sets status to 'failed' on unhandled errors.
