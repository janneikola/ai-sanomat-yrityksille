    **1. Auth plugin (api/src/plugins/auth.ts):**
    Use fastify-plugin to wrap @fastify/jwt and @fastify/cookie registration.
    - JWT secret from `process.env.JWT_SECRET`
    - Cookie config: `{ cookieName: 'token', signed: false }`
    - Decorate fastify with `authenticate` function that calls `request.jwtVerify()` and returns 401 on failure with `{ error: 'Unauthorized' }`
    - On server startup, hash `process.env.ADMIN_PASSWORD` with bcrypt (10 rounds) and store in memory as `adminPasswordHash`. Decorate fastify with it for login route access.

    **2. Auth route (api/src/routes/auth.ts):**
    FastifyPluginAsyncZod pattern. Register under `/api/auth` prefix (set in app.ts).
    - `POST /login`: Body schema `{ email: z.string().email(), password: z.string() }`. Check email === 'jannenne@gmail.com', bcrypt.compare password against `fastify.adminPasswordHash`. On success: sign JWT with `{ email, role: 'admin' }` expiring in 7d, set httpOnly cookie named 'token' with path '/', secure in production, sameSite 'lax', maxAge 7 days. Return `{ success: true }`. On failure: 401 `{ error: 'Virheelliset tunnukset' }`.
    - `POST /logout`: Clear the token cookie. Return `{ success: true }`.
    - `GET /me`: Protected with `onRequest: [fastify.authenticate]`. Return `{ email: request.user.email, role: request.user.role }`.

    **3. Services layer:**
    Each service file exports functions that use the drizzle db instance from `api/src/db/index.ts`.

    `api/src/services/clients.ts`:
    - `listClients()`: Select all from clients table, ordered by createdAt desc
    - `getClient(id)`: Select one by id
    - `createClient(data)`: Insert and return created row
    - `updateClient(id, data)`: Update by id, set updatedAt to new Date(), return updated row

    `api/src/services/sources.ts`:
    - `listSources()`: Select all from newsSources, ordered by createdAt desc
    - `getSource(id)`: Select one by id
    - `createSource(data)`: Insert and return created row
    - `updateSource(id, data)`: Update by id, return updated row
    - `toggleSource(id)`: Toggle isActive boolean, return updated row

    `api/src/services/templates.ts`:
    - `listTemplates()`: Select all from promptTemplates, ordered by name asc
    - `getTemplate(id)`: Select one by id
    - `updateTemplate(id, data)`: Update by id, return updated row
    (No createTemplate - templates are seeded only)

    **4. CRUD routes:**
    All admin routes use `onRequest: [fastify.authenticate]` for JWT protection.
    Use FastifyPluginAsyncZod pattern with Zod schemas from @ai-sanomat/shared.

    `api/src/routes/clients.ts` (prefix `/api/admin` set in app.ts):
    - `GET /clients`: List all clients. Response schema: array of clientResponseSchema.
    - `POST /clients`: Create client. Body: createClientSchema. Response 201: clientResponseSchema.
    - `GET /clients/:id`: Get single client. Params: `{ id: z.coerce.number() }`. Response: clientResponseSchema. 404 if not found.
    - `PUT /clients/:id`: Update client. Params + body: updateClientSchema. Response: clientResponseSchema. 404 if not found.

    `api/src/routes/sources.ts`:
    - `GET /sources`: List all sources. Response: array of sourceResponseSchema.
    - `POST /sources`: Create source. Body: createSourceSchema. Response 201: sourceResponseSchema.
    - `GET /sources/:id`: Get single source. Response: sourceResponseSchema.
    - `PUT /sources/:id`: Update source. Body: updateSourceSchema. Response: sourceResponseSchema.
    - `PATCH /sources/:id/toggle`: Toggle source active status. Response: sourceResponseSchema.

    `api/src/routes/templates.ts`:
    - `GET /templates`: List all templates. Response: array of templateResponseSchema.
    - `GET /templates/:id`: Get single template. Response: templateResponseSchema.
    - `PUT /templates/:id`: Update template. Body: updateTemplateSchema. Response: templateResponseSchema.

    - Add GET /health route returning `{ status: 'ok', timestamp: new Date().toISOString() }`
    - Export `buildApp` function

    **IMPORTANT:** Use `import { z } from 'zod/v4'` everywhere. Use FastifyPluginAsyncZod type from fastify-type-provider-zod. Error responses should use `{ error: string }` format consistently.
