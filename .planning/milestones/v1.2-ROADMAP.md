# Milestone v1.2: Newsletter Quality & Design

**Status:** ✅ SHIPPED 2026-03-04
**Phases:** 10-13
**Total Plans:** 4

## Overview

Make the HTML newsletter significantly better — structured content, relevant images, proper branding. Every story gets a lead sentence, bullet points, and visual hierarchy. Images come from source OG metadata first, with AI-generated infographic fallback. AI-Sanomat logo appears in every email header with dark mode protection.

## Milestone Summary

**Key Decisions:**
- Logo rendered above AI-Sanomat text heading, not replacing it
- White island uses #FAFAFA background for dark mode logo protection
- story.lead as structured-vs-fallback discriminator (not contentBlocks presence)
- Inline styles on ul/li for Outlook Word engine compatibility
- Buffer.byteLength for accurate Finnish multi-byte character size measurement
- toImageUrl extracted as module-level export with baseUrl parameter for testability
- PLACEHOLDER_IMAGE_URL eliminated; undefined used as fallback
- Gemini prompts only for stories without OG images (save API tokens)
- Hero image always generated via Gemini even when all stories have OG images

**Issues Resolved:**
- Broken image tags from PLACEHOLDER_IMAGE_URL (no actual placeholder file existed)
- OG image URLs being incorrectly prefixed with API base URL (absolute URL passthrough)

**Issues Deferred:**
- INT-01: Health dot key mismatch in X monitoring page (carry-over from v1.1)
- INT-02: Duplicate items not filtered during digest generation (carry-over from v1.1)
- Gemini billing verification needed for production image generation (free tier is 0 IPM)
- Litmus/Email on Acid integration for automated email testing

**Technical Debt Incurred:**
- Pre-existing TypeScript errors in deduplicationService.ts (embedding column references)
- Production prompt template requires manual update via admin UI (onConflictDoNothing in seed)
