  <action>
Create `web/src/app/(admin)/clients/[id]/page.tsx` -- a client detail page with the full digest generation, preview, approve/send workflow.

**Page structure:**

1. **Client info header:**
   - Fetch client data from `GET /api/admin/clients/${id}` (use the `id` from Next.js dynamic route params)
   - Display client name (h1), industry, contact email, plan
   - "Takaisin" (Back) link to /clients

2. **Digest generation section:**
   - "Generoi katsaus" (Generate digest) button
   - On click: `POST /api/admin/digests/generate` with `{ clientId: number }`
   - Show loading spinner (Loader2) during generation with text "Generoidaan katsausta..."
   - On success: store the returned `issueId` in state, fetch digest details
   - On error: show sonner toast with error message

3. **Latest digest display:**
   - Also fetch digests for this client on page load: `GET /api/admin/digests` and filter by clientId, take most recent
   - Show digest status badge (color-coded: ready=yellow, approved=blue, sent=green, failed=red, generating=gray)
   - Show week number and year

4. **Email preview (ADMIN-04):**
   - When a digest with status 'ready' or 'approved' is selected:
   - Display an `<iframe>` (width 100%, height 800px, border with rounded corners) with src pointing to `/api/admin/digests/${issueId}/preview`
   - The iframe renders the exact HTML email as recipients would see it
   - Add a "Esikatselu" (Preview) heading above the iframe

5. **Action buttons (ADMIN-05):**
   - When digest status is 'ready':
     - "Hyvaksy ja laheta" (Approve & Send) button (primary/default variant)
       - On click: first call `POST /api/admin/digests/${issueId}/approve`, then `POST /api/admin/digests/${issueId}/send`
       - Show loading state "Lahetetaan..."
       - On success: toast "Katsaus lahetetty {sent} vastaanottajalle" and refresh digest status
     - "Generoi uudelleen" (Regenerate) button (outline variant)
       - On click: `POST /api/admin/digests/${issueId}/regenerate`
       - On success: update with new issueId, refresh preview
   - When digest status is 'approved' (edge case -- approved but not yet sent):
     - "Laheta" (Send) button
     - "Generoi uudelleen" button
   - When digest status is 'sent':
     - Show "Lahetetty" (Sent) confirmation text with sentAt timestamp
     - No action buttons
   - When digest status is 'failed':
     - Show error indication
     - "Generoi uudelleen" button

6. **Validation report (optional display):**
   - If digest has validationReport, show a collapsible section with quality score and any flagged issues
   - Use a simple details/summary or Card with toggle

**Technical notes:**
- Use `'use client'` directive
- Get the dynamic `id` param via `useParams()` from `next/navigation` (Next.js App Router pattern)
- Use `useState` for digest, client, loading states
- Use `useEffect` for initial data loading
- Use sonner `toast` for success/error feedback
- Import shadcn/ui components: Button, Card, CardContent, CardHeader, CardTitle, Badge (if available, else use styled span)
