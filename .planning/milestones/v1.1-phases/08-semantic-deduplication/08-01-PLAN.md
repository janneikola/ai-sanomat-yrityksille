<objective>
Add semantic deduplication to the news pipeline using OpenAI text-embedding-3-small embeddings stored in PostgreSQL via pgvector, with cosine similarity search to detect and flag near-duplicate news items across all sources.

Purpose: Currently, news items are deduplicated only by URL (onConflictDoNothing on the url unique constraint). This misses the common case where the same AI story is reported by multiple sources (RSS, Beehiiv, web search) with different URLs but nearly identical content. Semantic deduplication catches these cross-source duplicates, improving digest quality by preventing repetitive content.

Output: OpenAI embedding client, deduplication service with two-tier similarity thresholds, schema changes for pgvector storage, post-collection embedding pipeline integrated into news collector, admin API routes for dedup review, and admin deduplication review page with override capability.
</objective>

    5. **Create deduplication service** (`api/src/services/deduplicationService.ts`):
       - Import from drizzle-orm: `eq`, `and`, `isNull`, `isNotNull`, `ne`, `desc`, `sql`, `gt`
       - Import `cosineDistance` from `drizzle-orm`
       - Import `db` from `../db/index.js`, `newsItems` from `../db/schema.js`
       - Import `generateEmbeddings` from `../integrations/openaiClient.js`
       - Constants: `EXACT_DUPLICATE_THRESHOLD = 0.95`, `NEAR_DUPLICATE_THRESHOLD = 0.85`, `DEDUP_WINDOW_DAYS = 14`, `BATCH_SIZE = 50`

       - **`export async function findSimilarItems(embedding: number[], excludeId: number, threshold = NEAR_DUPLICATE_THRESHOLD)`**:
         - Compute similarity: `const similarity = sql<number>\`1 - (${cosineDistance(newsItems.embedding, embedding)})\``
         - Query newsItems where: similarity > threshold, embedding is not null, id != excludeId, collectedAt > (now - DEDUP_WINDOW_DAYS)
         - Select: id, title, url, similarity
         - Order by similarity DESC, limit 5
         - Return the result array

       - **`export async function processNewEmbeddings(): Promise<{ embedded: number; duplicatesFound: number }>`**:
         - Fetch items without embeddings: `WHERE embedding IS NULL`, ordered by id ASC, limit BATCH_SIZE
         - If none found, return `{ embedded: 0, duplicatesFound: 0 }`
         - Build text array: `item.title + (item.summary ? ' ' + item.summary : '')` for each item
         - Call `generateEmbeddings(texts)` -- if all results are null (API key missing), return `{ embedded: 0, duplicatesFound: 0 }`
         - For each item with a non-null embedding:
           a. Update the item's embedding in DB: `db.update(newsItems).set({ embedding }).where(eq(newsItems.id, item.id))`
           b. Call `findSimilarItems(embedding, item.id)` to check for near-duplicates
           c. If similar items found:
              - Determine canonical: the item with the LOWEST id among the similar results (oldest = canonical)
              - If canonical.id < item.id (current item is newer), mark current item as duplicate:
                `db.update(newsItems).set({ isDuplicate: true, canonicalItemId: canonical.id }).where(eq(newsItems.id, item.id))`
              - If canonical.id > item.id (current item is older), mark the similar item as duplicate instead:
                `db.update(newsItems).set({ isDuplicate: true, canonicalItemId: item.id }).where(eq(newsItems.id, canonical.id))`
           d. Increment duplicatesFound counter
         - Return `{ embedded: count, duplicatesFound }`

       - **`export async function overrideDuplicate(itemId: number): Promise<boolean>`**:
         - Update the newsItem: set `isDuplicate = false`, `canonicalItemId = null`
         - Return true if rowCount > 0, false otherwise

       - **`export async function getDuplicates(limit = 100): Promise<Array<{...}>>`**:
         - Query newsItems where `isDuplicate = true`
         - For each duplicate, also fetch the canonical item (join or subquery)
         - Select: id, title, url, collectedAt, isDuplicate, canonicalItemId, and a computed similarity score (re-compute cosine distance between the item's embedding and the canonical item's embedding)
         - Also select canonical item's title and url (as canonicalTitle, canonicalUrl)
         - Order by collectedAt DESC, limit to the provided limit
         - If the join/subquery approach is complex, use two queries: first get duplicates, then batch-fetch canonical items by their IDs
