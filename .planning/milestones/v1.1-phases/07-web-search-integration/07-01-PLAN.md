<objective>
Integrate Tavily web search into the news collection pipeline so the system finds industry-specific AI news that RSS feeds miss, with per-client tailored queries and admin configuration UI.

Purpose: RSS/Beehiiv sources cover known AI publications but miss industry-specific AI applications and emerging trends. Tavily web search supplements this with targeted queries based on each client's industry, ensuring digests contain uniquely relevant content.

Output: Tavily integration client, web search service with caching and relevance filtering, modified news collector with client-aware web search loop, admin API routes, and admin web search management page.
</objective>

    2. **Extend DB schema** (`api/src/db/schema.ts`):
       - Add `'web_search'` to `sourceTypeEnum`: `pgEnum('source_type', ['rss', 'beehiiv', 'manual', 'web_search'])`
       - Add columns to `clients` table:
         - `webSearchEnabled: boolean('web_search_enabled').notNull().default(false)`
         - `searchPrompt: text('search_prompt')` (nullable -- null means auto-generate from industry)
         - `lastWebSearchAt: timestamp('last_web_search_at')` (nullable)
       - Add `searchCache` table:
         ```typescript
         export const searchCache = pgTable('search_cache', {
           id: serial('id').primaryKey(),
           queryHash: varchar('query_hash', { length: 255 }).notNull(),
           query: text('query').notNull(),
           clientId: integer('client_id').references(() => clients.id),
           results: text('results').notNull(), // JSON string of TavilyResult[]
           resultCount: integer('result_count').notNull().default(0),
           cachedAt: timestamp('cached_at').notNull().defaultNow(),
         });
         ```

    3. **Create Tavily client** (`api/src/integrations/tavilyClient.ts`):
       - Follow the same thin-wrapper pattern as `rssCollector.ts`
       - Import `tavily` from `@tavily/core`, initialize with `process.env.TAVILY_API_KEY`
       - Export `TavilyResult` interface: `{ title, url, content, score, publishedDate }`
       - Export `searchTavily(query: string, maxResults = 5): Promise<TavilyResult[]>` that calls `tvly.search()` with `topic: 'news'`, `searchDepth: 'basic'`, `timeRange: 'week'`, `maxResults`
       - If TAVILY_API_KEY is not set, log warning and return empty array (do NOT crash)

    4. **Create web search service** (`api/src/services/webSearchService.ts`):
       - **`generateSearchQueries(industry: string, customPrompt?: string | null): string[]`** -- Pure function. If customPrompt exists, use it as primary query plus industry variant. Otherwise auto-generate 3 queries: `AI ${industry} news`, `artificial intelligence ${industry} latest`, `${industry} tekoaly uutiset` (Finnish variant)
       - **`isAIRelevant(title: string, content: string): boolean`** -- Keyword filter. Check lowercase title+content for any of: `['ai', 'artificial intelligence', 'tekoaly', 'machine learning', 'koneoppiminen', 'neural', 'gpt', 'llm', 'generative', 'deep learning', 'syvaoppiminen', 'chatbot', 'automation', 'automaatio', 'robotiikka', 'robotics']`
       - **`isDueWithin24Hours(frequency, preferredDay, biweeklyWeek): boolean`** -- Check both today and tomorrow using existing `isDueToday()` from scheduleService. If either returns true, client is due.
       - **`searchWithCache(query: string, clientId: number): Promise<TavilyResult[]>`** -- Compute queryHash as `query.toLowerCase().trim()`. Check searchCache table for matching queryHash with cachedAt within last 24 hours. If cache hit, return parsed results. If miss, call `searchTavily()`, store in searchCache, return results.
       - **`searchForClient(clientId: number): Promise<{ collected: number; queries: number; cached: number }>`** -- Load client from DB. Generate queries. For each query, call `searchWithCache()`. Filter results with `isAIRelevant()`. Insert relevant results as news_items with sourceId of a shared "Web Search" source (find or create a news_source with type='web_search' name='Web Search'). Count collected, queries, cached. Update client's `lastWebSearchAt`. Apply health tracking (logFetchAttempt, updateSourceHealth) on the shared web search source.

    5. **Modify news collector** (`api/src/services/newsCollectorService.ts`):
       - After the existing source loop (after the for-of loop over sources), add a web search section:
       - Import `searchForClient` from `webSearchService.ts`
       - Query clients table for active clients with `webSearchEnabled=true`
       - Filter to clients due within 24 hours using `isDueWithin24Hours()` from webSearchService (skip if `schedulePaused`)
       - For each due client, call `searchForClient(client.id)` in a try/catch (same error isolation pattern as source loop)
       - Add web search results count to the return value

    6. **Update shared schemas** (`packages/shared/src/schemas/source.ts`):
       - Add `'web_search'` to all type enums in createSourceSchema, updateSourceSchema, and sourceResponseSchema

    7. **Update client schemas** (`packages/shared/src/schemas/client.ts`):
       - Add to `updateClientSchema`: `webSearchEnabled: z.boolean().optional()`, `searchPrompt: z.string().nullable().optional()`
       - Add to `clientResponseSchema`: `webSearchEnabled: z.boolean()`, `searchPrompt: z.string().nullable()`, `lastWebSearchAt: z.coerce.date().nullable()`
       - Update `createClientSchema` if needed to include `webSearchEnabled` and `searchPrompt`

    8. **Update clients service** (`api/src/services/clients.ts`):
       - Ensure the computed client response includes the new fields (`webSearchEnabled`, `searchPrompt`, `lastWebSearchAt`) from DB row

    9. **Create web search routes** (`api/src/routes/webSearch.ts`):
       - Follow existing route file pattern (export default Fastify plugin)
       - **GET `/web-search/clients`** -- Returns all clients with: id, name, industry, webSearchEnabled, searchPrompt, lastWebSearchAt. Also include most recent 5 searchCache entries per client (for recent results display).
       - **POST `/web-search/:clientId/trigger`** -- Calls `searchForClient(clientId)`. Returns `{ collected, queries, cached }`.
       - **PUT `/web-search/:clientId/config`** -- Updates client's `webSearchEnabled` and `searchPrompt` fields. Body: `{ webSearchEnabled?: boolean, searchPrompt?: string | null }`. Returns updated client.
       - All routes require admin auth (use `onRequest: [fastify.authenticate]`)

    10. **Register routes** (`api/src/app.ts`):
        - Import `webSearchRoutes` from `./routes/webSearch.js`
        - Register with `await app.register(webSearchRoutes, { prefix: '/api/admin' })` alongside other admin routes

    11. **Run db:push** to sync schema changes:
        ```bash
        cd api && npx drizzle-kit push
        ```

    Important notes:
    - The sourceTypeEnum modification requires `ALTER TYPE source_type ADD VALUE 'web_search'`. If `drizzle-kit push` fails on enum extension, run the ALTER TYPE manually before push.
    - Use `onConflictDoNothing()` on news_items inserts (existing URL unique constraint handles dedup across sub-queries).
    - Do NOT add clientId to news_items -- results go into the shared pool.
    - Use a single shared "Web Search" news_source row for health tracking (find or create it).
