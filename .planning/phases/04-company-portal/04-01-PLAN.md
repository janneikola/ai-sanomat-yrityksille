**Portal auth decorator** (defined inside the plugin, NOT exported):
```typescript
const authenticatePortal = async (request: FastifyRequest, reply: FastifyReply) => {
  try {
    await request.jwtVerify();
    if (request.user.role !== 'company') {
      return reply.code(403).send({ error: 'Forbidden' });
    }
  } catch {
    reply.code(401).send({ error: 'Unauthorized' });
  }
};
```

**Public routes (no auth):**

`POST /login` -- Request magic link:
- Body schema: `z.object({ email: z.string().email() })`
- Response 200: `z.object({ message: z.string() })` -- always "Jos tama sahkoposti on rekisteroity, saat kirjautumislinkin." (same message whether email exists or not -- per security pitfall from research)
- Call `generateMagicLink(fastify, email)` -- fire and forget (don't await to avoid timing attacks; or await but return same response regardless)
- Actually: await the call but catch errors silently (log them). Always return the generic success message.

`POST /verify` -- Verify magic link token:
- Body schema: `z.object({ token: z.string() })`
- Call `verifyMagicLink(fastify, token, reply)` -- returns `{ success, clientId }`
- On success: return `{ success: true, redirectUrl: '/tiimi' }`
- On failure (expired/invalid): return 401 `{ error: 'Linkki on vanhentunut tai virheellinen' }`

**Protected routes (authenticatePortal):**

`GET /members` -- List team members for logged-in company:
- `onRequest: [authenticatePortal]`
- Query: `db.select().from(members).where(eq(members.clientId, request.user.clientId!)).orderBy(members.createdAt)`
- Return array of members with id, email, name, isActive, isBounced

`POST /members` -- Add single member:
- `onRequest: [authenticatePortal]`
- Body: `z.object({ email: z.string().email(), name: z.string().optional() })`
- Check if member already exists for this clientId: if exists and inactive, reactivate (set isActive=true, isBounced=false). If exists and active, return 409 conflict. If not exists, insert.
- Return the created/reactivated member

`POST /members/bulk` -- Bulk import members:
- `onRequest: [authenticatePortal]`
- Body: `z.object({ emails: z.string() })` -- comma-separated string
- Parse: split by comma, trim whitespace, filter empty strings
- Validate each with `z.string().email()`, collect valid and invalid
- For each valid email: check existing (reactivate if inactive, skip if active)
- Return summary: `{ added: number, reactivated: number, skipped: number, invalid: string[] }`

`PATCH /members/:id` -- Soft-delete (deactivate) a member:
- `onRequest: [authenticatePortal]`
- Params: `z.object({ id: z.coerce.number() })`
- Verify member belongs to `request.user.clientId` -- 404 if not
- Set `isActive: false` (soft delete per user decision -- preserves delivery history)
- Return `{ success: true }`

`GET /archive` -- List sent newsletters for the company:
- `onRequest: [authenticatePortal]`
- Query: `db.select({ id, weekNumber, year, status, createdAt, updatedAt }).from(issues).where(and(eq(issues.clientId, request.user.clientId!), eq(issues.status, 'sent'))).orderBy(desc(issues.updatedAt))`
- Return array of sent issues

`GET /me` -- Current portal user info:
- `onRequest: [authenticatePortal]`
- Query client by `request.user.clientId` -- return `{ companyName, plan, contactEmail }`
- Used by portal layout to display company name and plan

`POST /logout` -- Clear cookie:
- `reply.clearCookie('token', { path: '/' }).send({ success: true })`

```typescript
async function verifyAndDecode(token: string): Promise<{ email: string; role: string; clientId?: number } | null> {
  try {
    const { payload } = await jwtVerify(token, secret);
    return payload as { email: string; role: string; clientId?: number };
  } catch {
    return null;
  }
}
```

Update the `middleware` function with this logic (ORDER MATTERS -- check portal routes BEFORE admin catch-all):

1. **Portal public pages** (`/portal/login`, `/portal/verify`):

2. **Portal protected pages** (`/tiimi`, `/arkisto`):
   - No token -> redirect to `/portal/login`
   - Token invalid -> redirect to `/portal/login`, clear cookie

4. **Admin pages** (everything else) -- existing logic unchanged:
